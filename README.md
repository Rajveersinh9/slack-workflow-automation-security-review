# Slack Workflow Automation & Security Review

## Overview

This project explores building structured Slack workflows and evaluating third-party Slack applications from a security and compliance perspective.

The objective was to:

- Design internal Slack workflow automation
- Improve IT notification routing
- Evaluate third-party Slack apps for risk
- Apply identity & access management principles

---

## Workflow Built

Created a simulated Slack workflow for IT ticket routing:

Trigger:
- Form submission for IT support request

Action:
- Automatically post ticket summary in #it-support channel
- Tag priority level
- Notify assigned support group

## Security Review Component

Evaluated Slack apps based on:

- OAuth permission scopes
- Message read/write access
- User data access
- Admin privilege requirements
- Data retention policies

---

## Risk Management Approach

- Principle of least privilege
- Admin approval workflow
- Quarterly access review
- Removal of unused integrations

---

## Key Concepts Applied

- Identity & Access Management (IAM)
- Multi-Factor Authentication (MFA)
- Third-party risk assessment
- Security-first IT operations mindset

---

## Outcome

Demonstrates understanding of collaboration tool governance, workflow automation, and security-conscious IT support practices.
